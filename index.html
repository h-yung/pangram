Pangram

  1. Provide the seven letters of the day in the inputs.
  2. Once you do so, you can generate possible strings. View the outputs by clicking to expand the details/summary box (indicated by the triangles).
  3. You can then guess in fragments of potential words to reduce the number of possible strings in the current outputs.

Approach notes

There are three main pieces approached with flaws, the biggest being performance: How do you narrow down on the many combinations possible with seven letters, where each letter appears at least once but the final word length is only known to be greater than or equal to seven?

Absent a sophisticated language processing approach for the time being, brute force leads to the following:

  1. Very rough logic: In English, the same letter is unlikely to be repeated consecutively three times. So we get the function narrowDown(string) that returns a Boolean.
  2. A pangram for a given letter set can easily exceed seven-letter length, but in theory, a recognizable word can be spotted within the first seven letters. Currently, this requires a human/manual assessment. But until the thousands of combinations can be reduced, this approach has been deprioritized. Implications: Pangrams that exceed a seven-letter count are currently out of scope of this program, a major flaw.
  3. The last step is currently throwing the resulting seven-lettered strings - ideally pared down with fragmentary guesses - at a dictionary. This is very unkind and poor use of an API, so that part is actually detached from the current filtering and string generation logic and requires a manual call from console. Also see previous point for caveat.

Background

The New York Times releases a new Spelling Bee round every day (turn over is probably based on Eastern Time Zone). With the seven given letters, players must put together valid English words. A valid word that uses all seven letters — repeated letters allowed — will have found the "pangram". Sometimes there is more than one pangram for a given set of letters.

The challenge

Non-subscribers get very limited chances to guess the pangram — too many of the shorter words and you get locked out.

Naturally, if you want to get the pangram, you will conserve your guesses or try to make ones that seem incrementally useful. Sometimes you will figure it out in the first instant when looking at the words. But that's not always the case.

The solution?

It is currently faster for an English-speaking human (sample size: 1) to find the pangram than the program as currently written, but the work is in progress.
